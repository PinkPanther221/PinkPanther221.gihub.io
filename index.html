用eclipse创建maven项目——配置篇

Posted on 2020-03-17

准备做一个Java的项目，没想到配置环境就配置了一个晚上（虽然早应该认识到这是常态）。先上各种环境配置，我使用的是Eclipse(2019-12)+Maven3.6.3+Tomcat7.0的配置。整个过程分为如下几步：

在eclipse里创建maven项目

以最新版本的eclipse为例，创建maven项目时的路径为File-->New-->Maven project，此时会出现一个下拉框， 在里面选中maven-archetype-webapp创建然后点击Finish即可。最新的1.4版本在我这里发布有问题，所以最终选了较为古早的1.0版本，然后命名为o2o。

创建之后，会出现两个warning，大意为当前运行环境与本机安装运行环境不同。初始化时出现的是J2SE-1.5。为了将它调整为当前的1.8，我们右键项目—>Properties--->Java Build Path--->Libraries, 点击JRE System Libraries后点击右边的Edit，在Execution environment一栏选中JRE1.8版本，保存，Apply，此时两个warnings就会消失。

将maven与Tomcat关联起来

右键点击项目，Properties--->Java Build Path--->Libraries---> Add Library，点击Server Runtime，里面会有支持的Tomcat版本。笔者选了7.0。

为Test建立source folder

右键点击项目，新建Source Folder，命名为src/test/resources，然后右键项目，Properties--->Java Build Path--->Sources，将src/test/java 的Output folder目录复制到src/test/resources的Output folder目录项下。

  • 如果出现了Test source folder... is not also used for main sources 的错误，观察src/test/java的 Contain test sources 栏，如果是Yes的话把它改成No，然后保存，即可解决错误。

把maven项目变成web项目

现在的项目并不是一个web项目，因为并不包含Dynamic Web Module. 然而当我们右键点击项目，Properties--->Project Facet时，发现里面空空如也。此时我们需要在Eclipse中安装更多的模块。点击顶栏的Help--->Install New Software,在Work with地址栏中键入kepler - http://download.eclipse.org/releases/kepler, 下拉将Web,XML……下的项全选安装后重启Eclipse。

重启后再次打开Project Facet会发现这个时候出现了Dynamic Web Module的选项。不要改变版本！！！3.0是最适配的！！！勾选这个模块和JavaScript后保存关闭对话框，然后到src--->main--->webapp--->WEB-INF--->web.xml中将版本号从原本的2.3改成3.0

启动Tomcat

在底栏的Server下右键Tomcat7.0服务器，点击Add or Remove添加当前项目，依次点击Clean和Start。打开浏览器，在地址栏键入localhost:8080/o2o即可看到欢迎页面。到此，maven项目的环境配置成功。
